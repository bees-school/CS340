How do I write programs that are readable, maintainable and adaptable?
I focus mainly on proper syntax first as a way to ensure the code is correct, followed by moduling the code correctly so that it
can be used to tie back into other code if necessary and to not have duplicate code anywhere. Lastly, I use a lot of comments to keep myself on track
and ensure I am not missing anything within the code for the project as a whole.
How do I approach a problem as a computer scientist?
I feel as if it is important to take the customers wants and needs into consideration heavily.
If the customer requests a certain action to have a certain outcome, then it is my job to ensure I can code what they need
and make sure it works properly for them. I think of it a lot like customer service in that sense.
What do computer scientists do and why does it matter?
Computer scientists are essentially the backbone of almost any modern company that has an app that their customers use.
We are responsibile for creating sensibile and easy code to both read and use. After we create the code for the program, there
are times when the company will hire someone else to maintain it or update it as necessary, so the code itself needs to be concise and
well commented so whoever works on it next can make sense of it and easily adapt it to what the customer is wanting done.
